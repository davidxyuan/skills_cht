# Agent Brief 格式

agent brief 是給 AFK agent 使用的工作說明。目標是讓 agent 不需要重新訪談 maintainer，就能開始實作或調查。

每個 brief 都必須具體、可驗證、可交付。

## 必要結構

~~~markdown
> *This was generated by AI during triage.*

## Agent Brief

### Problem

使用者或系統面臨的問題。描述外部可觀察行為，不要只說內部猜測。

### Expected Behavior

完成後應該出現的行為。

### Current Behavior

目前實際發生什麼。若 bug 已重現，寫明重現結果。

### Relevant Context

- 相關 domain terms
- 相關檔案、modules、interfaces
- 既有測試或相似實作
- triage 時已排除的方向

### Suggested Approach

建議 agent 從哪裡開始。用 TDD 或 tracer bullet 切片描述，不要過度指定內部實作。

### Acceptance Criteria

- criterion 1
- criterion 2

### Test Plan

- 要新增或更新哪些行為測試
- 要跑哪些既有測試或檢查
~~~

## 原則

- 寫外部行為，不寫微觀實作命令。
- 保留 maintainer 已做出的決策。
- 若仍有開放問題，不要標成 ready-for-agent。
- brief 要能在幾天後仍然讀得懂。
- 測試應驗證 public interface 的行為，不要綁死 private implementation。
