# Skills 繁體中文速查表

這份速查表整理 skills_cht repo 中的 21 個繁體中文 skills。所有 skill 名稱都使用 -cht 後綴，方便和英文版共存。

## 使用範例

~~~text
請使用 $tdd-cht 幫我修這個 bug：曝光時間為 0 時量測會崩潰。先寫測試，再實作。
~~~

~~~text
請使用 $improve-codebase-architecture-cht 檢查目前專案，找出最值得重構的模組。
~~~

## Skills 清單

| Skill | 中文用途 |
| --- | --- |
| $caveman-cht | 超精簡溝通模式，省 token，去掉客套和廢話，但保留技術準確性。 |
| $design-an-interface-cht | 幫某個模組設計多種不同 API 或 interface 方案，適合比較架構選項。 |
| $domain-model-cht | 檢查計畫是否符合現有 domain model，釐清術語、決策與文件。 |
| $edit-article-cht | 編輯文章草稿，改善結構、清晰度、語氣和文字精煉度。 |
| $git-guardrails-claude-code-cht | 替 Claude Code 設 git 安全 hook，阻擋危險指令如 push、reset hard、clean。 |
| $github-triage-cht | 用 label 狀態機整理 GitHub issues，適合 issue 分流、bug 或 feature request 管理。 |
| $grill-me-cht | 像嚴格訪談一樣追問你的計畫或設計，逼出模糊點與決策分支。 |
| $improve-codebase-architecture-cht | 找出 codebase 架構改善機會，讓模組更深、更可測、更容易被 AI 理解。 |
| $migrate-to-shoehorn-cht | 把測試裡的 TypeScript as 斷言改成 @total-typescript/shoehorn。 |
| $obsidian-vault-cht | 搜尋、建立、整理 Obsidian 筆記，支援 wikilinks 和索引筆記。 |
| $qa-cht | 互動式 QA 回報流程：你描述 bug，它探索程式碼並建立 GitHub issue。 |
| $request-refactor-plan-cht | 訪談後產生細粒度 refactor plan，並整理成 GitHub issue。 |
| $scaffold-exercises-cht | 建立課程練習結構，包含 sections、problems、solutions、explainers。 |
| $setup-pre-commit-cht | 在 repo 設 Husky pre-commit hooks、lint-staged、Prettier、型別檢查和測試。 |
| $tdd-cht | 用 TDD red-green-refactor 流程開發功能或修 bug，先測試再實作。 |
| $to-issues-cht | 把 plan、spec 或 PRD 拆成可獨立執行的 GitHub issues。 |
| $to-prd-cht | 把目前對話和需求整理成 PRD，並準備成 GitHub issue。 |
| $triage-issue-cht | 探索 bug 根因，然後建立含 TDD 修復計畫的 GitHub issue。 |
| $ubiquitous-language-cht | 從對話中整理 DDD ubiquitous language 詞彙表，釐清領域術語。 |
| $write-a-skill-cht | 建立新的 agent skill，包含正確結構、說明和附加資源。 |
| $zoom-out-cht | 要求 agent 拉高視角，解釋某段程式碼在整體系統中的位置。 |
